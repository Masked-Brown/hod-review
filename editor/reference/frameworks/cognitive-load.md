## The claim in one paragraph

Cognitive Load Theory starts from one well-established fact about human memory: when we
deal with *novel* information, working memory is sharply limited — it can hold only a
handful of separate elements, and only for a short time — while long-term memory is, for
practical purposes, unlimited (Sweller, 1988; Sweller, Van Merriënboer & Paas, 1998). What
turns a beginner into an expert is not a bigger working memory but the schemas stored in
long-term memory that let many elements be handled as one. The instructional consequence is
blunt: a lesson that forces a novice to juggle more separate elements than working memory
can hold does not merely feel hard — it prevents the learning it intends, because the
capacity needed to build the schema has been spent elsewhere. Good design manages load; it
does not just add content.

## The three loads (in our words)

Cognitive Load Theory splits the demand a task places on working memory into three parts.

- **Intrinsic load** — the difficulty inherent in the material itself for *this* learner,
  set by how many elements interact and must be understood together ("element
  interactivity"). A vocabulary list is low; a multi-step process where each step depends on
  the last is high. It cannot be removed, but it can be *sequenced* — broken into parts and
  built up.
- **Extraneous load** — the demand added by *how the material is presented* rather than by
  the material itself. A diagram whose labels sit in a separate key, a slide read aloud
  verbatim while students also read it, a wall of text around one key idea: all spend
  working memory on the packaging instead of the content. This is the load an editor hunts
  for, because it is the load design can cut.
- **Germane processing** — the working-memory effort that actually goes into building and
  automating the schema. In the original 1998 framing this was treated as a third,
  desirable "load"; the 2019 restatement folds it into intrinsic processing rather than a
  separate additive source (Sweller, Van Merriënboer & Paas, 2019). Either way the design
  goal is the same: cut extraneous load so the freed capacity is available for the learning
  itself.

## The classic effects a slide can trip

These are the named, replicated consequences of mismanaging load — each is a handle a
finding can cite:

- **Worked-example effect.** Novices learn more from studying worked examples than from
  solving equivalent problems unaided; problem-solving search consumes the capacity schema-
  building needs.
- **Split-attention effect.** When two sources that must be integrated to make sense (a
  diagram and its separate labels; a graph and a caption elsewhere) are physically apart,
  the learner spends load holding one while finding the other. Integrating them removes it.
- **Redundancy effect.** Presenting the same information in two full forms at once — reading
  a slide aloud word-for-word, or a self-explanatory diagram plus a paragraph restating it —
  *adds* load rather than reinforcing. More is not safer.
- **Modality effect.** Some information is better carried partly by narration and partly by
  visuals than by piling everything into the visual channel, because working memory has
  partly separate capacity for verbal and visual material.
- **Expertise-reversal effect.** Supports that help a novice (heavy scaffolding, full worked
  examples) become redundant load for someone who has the schema. Guidance that is right for
  a beginner can hurt an expert — so "load" is always relative to *this* class.

## What it looks like when a slide gets it wrong

Generic shapes, not worked findings (those are in `examples.md`, M5):

- A slide **carrying six new terms, a diagram, and a paragraph of prose** all at once (high
  extraneous load stacked on high intrinsic load).
- A **diagram whose labels are in a separate key or a later slide** (split attention).
- A slide the teacher plans to **read aloud verbatim** while it is also on screen for
  students to read (redundancy).
- A **problem set given before any worked model** for a genuinely novel procedure (missing
  the worked-example effect).
- Scaffolding **kept identical for a top set who no longer need it** (expertise reversal).

The editor points at the overload and asks how the teacher will stage or strip it. It never
redraws the slide.

## Citable handles (what a finding names under `PRINCIPLE`)

| Handle | Flags |
|---|---|
| `CLT — working-memory limits` | Too many novel elements held at once |
| `CLT — intrinsic load / element interactivity` | Interacting material not sequenced |
| `CLT — extraneous load` | Presentation adds avoidable demand |
| `CLT — worked-example effect` | Unaided problem-solving where a model was needed |
| `CLT — split-attention effect` | Sources that must integrate are separated |
| `CLT — redundancy effect` | The same content duplicated in full |
| `CLT — modality effect` | Everything forced into one channel |
| `CLT — expertise-reversal effect` | Scaffolding mismatched to the class's expertise |

## Boundaries — honest caveats

- Load is **relative to the learner**. Nothing is "high load" in the abstract; the same
  slide can be right for one class and overwhelming for another. A finding must say why it
  is too much *for this class*, given the intake context — not assert overload in general.
- **Cutting load is not the goal; learning is.** Stripping a slide until it is trivial
  lowers intrinsic load past the point of usefulness. The theory argues for managing load
  toward schema-building, not minimising it.
- Measuring load precisely (dual-task, subjective scales) is a research activity; an editor
  reads it *qualitatively* from the slide. The claim is directional — "this stacks more than
  a novice can hold" — not a measured figure.
